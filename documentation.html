Why this project? Managing personal finances is a universal need. This app demonstrates real-world frontend engineering — responsive UI, data visualization, full CRUD operations, import/export, PWA installation, and state management. A deliberately client-side architecture was chosen: financial data is sensitive, so it stays entirely in the user’s browser — no server, no data collection, no privacy risk.

🌟 Project Highlights

A production-ready finance app built on intentional simplicity — pure HTML, CSS, and JavaScript, with Chart.js as the only external dependency. A conscious design decision to demonstrate that strong fundamentals outperform framework dependency.

🛠

Clean Architecture

Structured like a real-world app:

  • SDLC followed — Plan → Design → Build → Test
  • Modular files — HTML, CSS, JS, manifest, Service Worker
  • Full CRUD — Create, Read, Update, Delete transactions
  • Single responsibility per function — clean, readable code
  • Input validation at every form boundary
🛡️

Security & Privacy First

Built with ethical, secure practices:

  • XSS protection — escapeHtml() on all user inputs
  • 100% offline — all data stays in localStorage
  • No tracking, no analytics, no third-party calls
  • No login required — zero data collection
  • AI-assisted development using GitHub Copilot responsibly
🚀

Modern Web APIs

Uses the full power of the browser:

  • Chart.js — 4 real-time interactive charts
  • PWA — manifest.json + Service Worker for offline use
  • FileReader & Blob APIs — CSV import & export
  • Notification API — bill due date reminders
  • localStorage — offline-first, privacy-first persistent data
Bottom line: This project demonstrates that with strong fundamentals — clean architecture, security awareness, and modern browser APIs — you can ship a product that rivals paid fintech tools. The absence of a framework is not a limitation — it’s proof that the developer understands how the web actually works.

🤖 AI Next Month Prediction

A 3-month rolling average model that predicts next month’s total expenditure based on your recent spending history. No external API — 100% client-side.

FactorDetail
Input DataExpenses from the last 3 complete calendar months
AlgorithmRolling average: (Month-1 + Month-2 + Month-3) ÷ count of months with data
Minimum DataRequires at least 2 months of expense data — card hidden otherwise
DisplayPredicted amount + percentage of income (green <50%, yellow <80%, red ≥80%)
Edge CasesMonths with zero expenses are excluded from average to prevent skew
UpdatesRecalculates on every transaction add/edit/delete
Why rolling average? It is simple, transparent, and requires no server. The 3-month window balances recency with stability — a one-month spike won’t break the prediction. This is the same technique used by Excel forecasting and Power BI trend analysis.

💾 Data Persistence

All data is saved in the browser's localStorage under 6 separate keys:

fin_transactions

Array of all transaction objects — description, amount, type, category, date, notes, recurring flag.

fin_budget_limits

Object mapping expense category → monthly budget limit. User-edited values override defaults.

fin_categories

Object with income[] and expense[] arrays of category names. Includes user-added custom categories.

fin_reminders

Array of reminder objects — title, amount, due date, and done status.

fin_goals

Array of savings goal objects — name, target amount, deadline, and creation date. Progress calculated from live net balance.

fin_theme

String — light or dark. Theme preference persists across page reloads automatically.

// Loading all state on startup let transactions = JSON.parse(localStorage.getItem('fin_transactions') || '[]'); let customBudgets = JSON.parse(localStorage.getItem('fin_budget_limits') || 'null') || { ...DEFAULT_BUDGET_LIMITS }; let customCats = JSON.parse(localStorage.getItem('fin_categories') || 'null') || { income: [...DEFAULT_CATEGORIES.income], expense: [...DEFAULT_CATEGORIES.expense] }; let reminders = JSON.parse(localStorage.getItem('fin_reminders') || '[]');

Financial Health Score algorithm:

// Component 1: Savings rate (max 40 pts) if (rate >= 30) score += 40; else if (rate >= 20) score += 30; else if (rate >= 10) score += 20; else if (rate > 0) score += 10; // Component 2: Budget adherence (max 30 pts) const adherence = (totalCats - overBudgetCount) / totalCats; score += Math.round(adherence * 30); // Component 3: Has income recorded (15 pts) if (income > 0) score += 15; // Component 4: Transaction count (max 15 pts) if (count >= 20) score += 15; else if (count >= 10) score += 10; else if (count >= 5) score += 5;

CSV Import — custom parser (handles quoted fields):

function parseCSVLine(line) { const result = []; let cur = ''; let inQ = false; for (let i = 0; i < line.length; i++) { if (line[i] === '"') { inQ = !inQ; } else if (line[i] === ',' && !inQ) { result.push(cur); cur = ''; } else { cur += line[i]; } } result.push(cur); return result; }

🔒 Security

🛡

XSS Prevention

All user inputs (desc, notes, category names) pass through escapeHtml() before rendering to DOM — prevents Cross-Site Scripting. Non-string values are cast via String() before escaping.

🔐

No External Data Transfer

All data stays in the user's browser localStorage. No servers, no analytics, no third-party requests — complete privacy.

✓

Input Validation

Amount must be a positive number, description is required, date is required — validated before any processing. CSV import validates each row individually.

🔗

Safe URL Handling

Blob URLs for CSV export are revoked immediately after download using URL.revokeObjectURL() — no memory leaks or dangling references.

📋

CSV Injection Prevention

Export wraps all values in double quotes and escapes internal quotes — prevents formula injection attacks when the file is opened in Excel or Google Sheets.

🔔

Permission-Based Notifications

Browser Notifications only fire after explicit user permission. Denied state is handled gracefully without errors or repeated prompts.
